# 地图几何与导航校验

## 数据源

地图数据由真实模型、平面资料和语义拓扑共同约束。模型提供建筑轮廓和三维质感，DWG/SKP/参考图提供空间理解和房间语义，前端数据结构提供可导航节点和可交互对象。

| 数据 | 作用 |
| --- | --- |
| `jingong.glb` | 主 3D 模型 |
| `jingong-fallback.glb` | fallback 模型 |
| `mapData.ts` | 房间、空间、门洞、楼梯、墙体、中心线、校准点 |
| `routeService.ts` | 路线搜索、距离、时间和分段结果 |
| `runtime.ts` | 图层、标签、路线文案和移动端运行时策略 |
| `modelAlignment.ts` | 模型坐标、楼层高度、202 平台和默认相机 |

## 空间对象

| 对象 | 校验要求 |
| --- | --- |
| 房间 | 每个房间有唯一 ID、名称、楼层、polygon、中心点和门节点 |
| 门洞 | 门洞为线段，门点位于线段中心并落在房间边界 |
| 走廊 | 走廊面和中心线保持一致，路线沿中心线通行 |
| 楼梯 | 公共楼梯和内部楼梯分别建模，landing 与导航节点配对 |
| 服务空间 | 卫生间、服务间、仓储、预留空间和空置空间有独立分类 |
| 二层半 | `202-5` 使用独立平台高度和二层承托结构 |

## 路线约束

| 路线 | 必经节点 |
| --- | --- |
| `101 -> 104-2F01` | `center-101`、`door-101`、`stair-104-1f`、`stair-104-2f`、`door-104-2F01`、`center-104-2F01` |
| `101 -> 108-2F04` | `center-101`、`door-101`、`stair-108-1f`、`stair-108-2f`、`door-108-2F04`、`center-108-2F04` |
| `108-lobby -> 202-5` | `center-108-lobby`、`door-108-lobby`、`stair-public-1f`、`stair-public-2f`、`c2-202`、`door-202-5`、`center-202-5` |

公共楼梯禁止直连 `104-2F01 / 106-2F / 108-2F04`。内部楼梯边必须标记为 `internal-stair`，公共楼梯边必须标记为 `stair`。

## 校准门禁

`verify-alignment.mjs` 检查模型资产、控制点、门洞数量、楼层高度、语义 bbox 和模型缩放。当前发布记录中控制点误差进入门禁范围，门洞映射、路线约束和空间分类通过。

| --- | --- |
| 房间 | 53 |
| 门洞线段 | 53 |
| 空间 | 72 |
| 中心线 | 16 |
| 主模型 | 47 meshes / 5000 vertices |
| 对齐误差 | max 0.074 / avg 0.037 记录于 Android smoke v36 |

## 视觉校验

| 视图 | 检查点 |
| --- | --- |
| 全楼总览 | 上下层可分辨，路线和终点不被遮挡 |
| 一层 | 房间、走廊、门、楼梯和墙体清晰 |
| 二层 | 二层墙体和承托结构完整，独立二层不误连公共楼梯 |
| 202 平台 | 二层半平台、公共楼梯和下方承托同时可理解 |
| 分层总览 | 楼层间距足够，楼梯上下口能配对 |
| 路线视角 | 当前点、下一门洞/楼梯、终点 pin 明确 |
